# Система кодирования сущностей онтологии курса химии

Версия: 1.0\
Назначение: единая, непротиворечивая система идентификаторов (ID) для
всех сущностей knowledge graph курса химии.

------------------------------------------------------------------------

# 1. Цели системы

1.  ASCII-only идентификаторы (без Unicode).
2.  Стабильность ключей (можно использовать как ключ словаря, URL,
    индекс).
3.  Машиночитаемость и отсутствие двусмысленности.
4.  Поддержка фаз, концентраций, гидратов, комплексов.
5.  Совместимость с существующими файлами:
    -   elements.json
    -   ions.json
    -   reactions.json

------------------------------------------------------------------------

# 2. Общие правила для всех ID

## 2.1 Разрешённые символы

Базовые: - a--z - 0--9 - \_

Дополнительно для chemical formula внутри substance_id: - A--Z - ( ) -
\[ \] - \*

Запрещено: - пробелы - кириллица - юникодные индексы (H₂SO₄ → H2SO4) -
знаки ±, ↑, ↓, → и т.п.

------------------------------------------------------------------------

## 2.2 Пространства имён (Namespace Prefixes)

Каждый ID начинается с префикса типа:

  Тип сущности   Префикс
  -------------- ---------
  Element        el:
  Ion            ion:
  Substance      sub:
  Reaction       rx:
  Process        proc:
  Scenario       sc:
  Skill          sk:
  Property       prop:
  Constraint     cons:
  Quantity       q:
  Unit           unit:
  Measurement    meas:
  Problem        prob:

Пример:

    sub:NaCl_aq
    rx:neutral_001
    proc:heating
    sk:oxidation_state

------------------------------------------------------------------------

# 3. Elements

Формат:

    el:<symbol_lowercase>

Примеры:

    el:h
    el:na
    el:fe
    el:cu

------------------------------------------------------------------------

# 4. Ions

Формат:

    ion:<formula>_<charge>

Примеры:

    ion:H_plus
    ion:Ca_2plus
    ion:SO4_2minus
    ion:Fe_3plus

------------------------------------------------------------------------

# 5. Substances

Формат:

    sub:<ASCII_FORMULA>_<phase>[_<modifier>]

## 5.1 Фазы (обязательный суффикс)

  Фаза                 Суффикс
  -------------------- ---------
  aqueous              \_aq
  solid                \_s
  liquid               \_l
  gas                  \_g
  precipitate (опц.)   \_ppt

Примеры:

    sub:H2O_l
    sub:NaCl_aq
    sub:CaCO3_s
    sub:CO2_g

------------------------------------------------------------------------

## 5.2 Концентрация (если влияет на химию)

    sub:H2SO4_aq_dil
    sub:H2SO4_aq_conc

------------------------------------------------------------------------

## 5.3 Гидраты

    sub:CuSO4*5H2O_s

------------------------------------------------------------------------

## 5.4 Комплексы

    sub:Na[Al(OH)4]_aq

------------------------------------------------------------------------

# 6. Reactions

Формат:

    rx:<type>_<short_description>_<index>

Примеры:

    rx:neutral_hcl_naoh_001
    rx:precip_baso4_001
    rx:redox_kmno4_fe2_001
    rx:decomp_caco3_heat_001

------------------------------------------------------------------------

# 7. Processes

Формат:

    proc:<name>

Примеры:

    proc:heating
    proc:dissolution
    proc:precipitation
    proc:gas_evolution
    proc:stirring
    proc:dilution
    proc:filtration

------------------------------------------------------------------------

# 8. Scenario

Формат:

    sc:<topic>_<index>

------------------------------------------------------------------------

# 9. Skills

Формат:

    sk:<concept_name>

------------------------------------------------------------------------

# 10. Properties

Формат:

    prop:<name>

------------------------------------------------------------------------

# 11. Constraint

Формат:

    cons:<name>

------------------------------------------------------------------------

# 12. Итог

Система: - строгая, - машиночитаемая, - графо-ориентированная, -
пригодная для энергетических расчётов, - масштабируемая.
