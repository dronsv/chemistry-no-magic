# Короткий план действий

## Ближайшая цель

Улучшить качество text realization без запуска большого архитектурного проекта.

## Шаг 1. Реализовать `decline()`

### Результат
- rule-based склонение для типовых single-word терминов;
- overrides для irregular cases;
- интеграция в существующий renderer.

### Польза
- меньше ручных форм;
- меньше дублирования в overlays;
- лучше поддерживаемость.

## Шаг 2. Довести Phase B1 по rule summaries

### Результат
- генерация summary texts из structured rule data;
- единообразие формулировок;
- меньше свободного text drift.

### Польза
- практический construction layer без framework overbuild.

## Шаг 3. Добавить только нужные morph directives

### Результат
- точечное усиление prompt templates;
- покрытие реальных падежных и number use-cases.

### Польза
- эволюция без переписывания template system.

## Шаг 4. Провести inventory шаблонов

### Что зафиксировать
- какие template families уже существуют;
- какие intention-types уже фактически покрыты;
- где есть дублирование;
- где не хватает параметризации.

### Польза
- станет понятно, нужен ли вообще отдельный intention layer позже.

## Чего не делать на этом этапе

- не запускать отдельную architecture freeze фазу;
- не строить новый AST ради чистоты модели;
- не делать универсальный construction framework;
- не выносить discourse planner.

## Критерий успеха

Через 1–2 итерации должно стать лучше в трёх местах:

- меньше ручной морфологии;
- лучше rule text generation;
- больше выразительности у существующих templates.

Если этого достаточно, значит выбранный путь был верным и дальнейшая формализация не требуется немедленно.
